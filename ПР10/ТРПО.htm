Практическая работа 10. Разработка справочной системы.
Цель работы: Ознакомиться с особенностями технологии разработки справочной системы.

Предисловие.

Даже при наличии соответствующих инструментов разработка справочника является самостоятельной инженерно-технической задачей. По сложности такая задача соизмерима, пожалуй, с разработкой веб-сайта. Да и порядок действий при создании справочника во многом напоминает работу над веб-публикацией.

Порядок выполнения работы:

  1. Изучить дополнительные материалы по вопросам изучения технологии создания справочной системы программного продукта.
  2. Создать необходимые страницы (HTML-документы) справочной системы для вашей индивидуальной ПС. ( Создание HTML-документов можно выполнить с помощью приложения Блокнот).
  3. Сдать и защитить работу.
  4. Готовые материалы сохранить в папке Рабочий стол\Документы-stud\П <номер группы>\ pr_10_fio.

Защита отчета по практической работе

Защита отчета по лабораторной работе заключается в предъявлении преподавателю полученных результатов (на экране монитора), демонстрации полученных навыков и ответах на вопросы преподавателя.

Контрольные вопросы:

  1. Какую роль играет справочник в работе программных продуктов?
  2. Почему необходимо сопровождать Справку наряду с программным продуктом?
  3. Каковы особенности создания справочных файлов в приложении CHM Builder ?
